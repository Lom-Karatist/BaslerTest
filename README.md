# BaslerTest

![Логотип проекта](Icons/hals.png)

**BaslerTest** — тестовое приложение для управления камерами Basler (модель acA1920-40um) в режиме master-slave с аппаратной синхронизацией. Проект разрабатывается как модуль для будущей гиперспектральной системы ГАЛС, но может использоваться самостоятельно для отладки и изучения работы с камерами Basler.

## 🚀 Возможности

- **Работа с двумя камерами** в режиме master-slave через аппаратную синхронизацию (Line3 → Line4).
- **Асинхронный захват изображений** с использованием `QThreadPool` и `QRunnable` — камеры работают в отдельных потоках.
- **Динамическое изменение параметров** без остановки потока:
  - Экспозиция (мс)
  - Усиление (Gain)
  - Частота кадров (FPS)
  - Формат пикселя (Mono8, Mono12, Mono12p)
  - Размер изображения (ширина, высота)
  - Смещение области интереса (Offset X/Y)
  - Биннинг (коэффициенты 1–4, режимы Sum/Average)
- **Автоматическая корректировка связанных параметров**:
  - Экспозиция и частота кадров согласуются (приоритет у частоты кадров).
  - Размер, смещение и биннинг пересчитываются для сохранения физического размера области.
  - Смещение автоматически приводится к кратности 4 (требование камеры).
- **Графический интерфейс** с отдельными формами для каждой камеры (гиперспектрометр и обзорная камера).
- **Сохранение настроек** в INI-файлы (`HS.ini`, `OC.ini`) с мгновенной записью.
- **Версионирование** через `version.h` и ресурсный файл Windows (`.rc`).

## 🛠️ Стек технологий

- **Qt 6** (модули: Core, GUI, Widgets, Concurrent, Svg)
- **Pylon SDK** (версия 6.3.0) для работы с камерами Basler
- Компилятор **MinGW** или **MSVC** (Windows)
- Планируется портирование на **Linux (Arm64)** для Raspberry Pi 5

## 📦 Сборка и запуск

### Требования
- Установленный Qt 6 (с компонентами для вашего компилятора)
- Pylon SDK 6.3.0 (для Windows)
- Камеры Basler acA1920-40um (или совместимые)

### Инструкция

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/Lom-Karatist/BaslerTest.git
   ```
2. Откройте файл проекта `BaslerTest.pro` в Qt Creator.
3. Убедитесь, что пути к Pylon SDK указаны корректно (см. `Basler.pri`).
4. Настройте сборку под вашу платформу (Windows 64-bit).
5. Соберите проект и запустите.

**Важно:** Перед запуском убедитесь, что камеры подключены к USB 3.0 и соединены кабелем синхронизации (пин 3 → пин 4).

## 📁 Структура проекта

- `BaslerApi.h/cpp` — основной класс для работы с камерой в потоке.
- `CameraManager.h/cpp` — управление двумя камерами, координация настроек.
- `BaslerSettings.h/cpp` — загрузка/сохранение параметров в INI.
- `BaslerSettingsForm.h/cpp/ui` — виджет настроек для одной камеры.
- `BaslerWindow.h/cpp/ui` — главное окно приложения.
- `Basler.pri` — подключаемый файл для интеграции модуля в другие проекты.
- `version.h`, `resource.rc`, `hals.ico` — ресурсы Windows и версионирование.
